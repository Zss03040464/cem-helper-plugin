# 深度改进说明

本仓库在原有脱敏示例的基础上进行了以下改进：

## 1. 模块化规则引擎

* 将规则定义抽象为 `platformDefaults` 与 `fieldAliases`，便于新增平台和字段别名。
* 新增 `fuzzyMatch` 函数，支持通过字段名称的子串模糊匹配别名，并根据匹配长度估算置信度。
* 引入 `generateSuggestions` 批量处理接口，方便浏览器扩展等调用。

## 2. 可扩展脱敏策略

* 将敏感模式抽象为数组 `patterns`，每个模式包含名称、正则表达式和替换函数。
* 新增身份证号、邮箱、地址等示例脱敏规则，可根据需求继续扩展。
* 提供 `detectSensitive` 帮助在发布前扫描潜在泄露。

## 3. 浏览器扩展的审核面板

* 在 `extension` 中的 content 脚本中，自动填充前会根据置信度决定是否需要人工审核。
* 如果字段已有值或置信度不足，扩展会生成待审核列表供用户确认后再填入，避免误填。

## 4. 审计与测试

* 提供 `scripts/audit-sanitization.js`，自动扫描项目中是否包含禁止提交的关键词或敏感模式。
* 使用 `node --test` 在 `test` 目录中添加测试用例，确保规则引擎和脱敏逻辑在修改后仍然正确。

## 5. 文档完善

* 在 `README.md` 中新增快速上手、特性说明和提交注意事项。
* 本文档详细列出改进内容，为后续贡献者提供参考。
